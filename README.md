# Git-Eval S ランク: Specialist

技術選定にうるさいレベル。最高ランクです。

## お題: 最適な技術選定で問題を解決しよう

目的に対して最適な手段を選び、その根拠を示してください。
目的と手段の逆転には厳しく減点します。

### やること

1. 実用的なプロジェクトを作る
2. `git-eval-config.json` を設定する
3. `DECISION.md` に設計判断を書く（**強く推奨**）
4. テスト・リンターを通す
5. commit して push する

### DECISION.md に書くべきこと

- なぜこの技術スタックを選んだか（代替案との比較）
- 目的に対して手段が過不足ないか
- パフォーマンス・スケーラビリティの考慮
- 独自のアプローチがある場合はその説明

### 評価ポイント（100 点満点）

| チェック項目 | 配点 | 評価方法 |
|---|---|---|
| テスト・リンター通過 | 10 | CI/CD |
| 技術選定の適切さ | 30 | LLM |
| 目的と手段の一致 | 25 | LLM |
| 創意工夫・最適化 | 20 | LLM |
| 総合力 | 15 | LLM |

### 注意

- オーバーエンジニアリングは減点対象です
- 「使ってみたかったから」は理由になりません
- 適切なシンプルさは高評価です
